Problema 1 y 5

¿Cuál es el comando para cambiar el nombre del archivo algebra1.txt a algebra2.txt en Linux?

Solución

mv algebra1.txt algebra2.txt

Explicación

El comando mv (move) se utiliza tanto para mover archivos como para renombrarlos. Si la ubicación de destino es el mismo directorio, el comando simplemente cambia el nombre del archivo.

Problema 2

¿Cuál es el resultado de este comando? echo "Esto es un ejemplo de tr para com-99" | tr 'aeiou' 'AEIOU'

Solución

EstO Es Un EjEmplO dE tr pArA cOm-99

Explicación

El comando echo envía el texto al comando tr a través de una tubería (|). El comando tr (translate) reemplaza cada carácter del primer conjunto ('aeiou') por su correspondiente en el segundo conjunto ('AEIOU'), convirtiendo solo las vocales a mayúsculas.

Problema 3

¿Cómo se puede crear un nuevo directorio en Linux utilizando el comando "mkdir" y cómo se puede crear una jerarquía de directorios en una sola línea de comando?

Solución

Crear un solo directorio:

mkdir mi_directorio

Crear una jerarquía (varios directorios anidados a la vez):

mkdir -p dir1/dir2/dir3

Explicación

mkdir crea directorios. La opción -p (parents) le indica a mkdir que cree también los directorios padres necesarios en la ruta si estos no existen.

Problema 4

Dado el archivo texto.txt con el contenido:
la utilidad cut
es fácil de usar y es útil
¿Cómo sería el comando para que me muestre:
la tili
es áci

Solución

cut -c 1-2,5-8 texto.txt

Explicación

El comando cut se usa para extraer secciones de cada línea. La opción -c especifica que se cortará por posición de caracteres. 1-2 extrae los dos primeros caracteres y 5-8 extrae los caracteres desde la quinta hasta la octava posición en cada línea.

Problema 6

¿Qué tipo de permiso no corresponde según el siguiente comando? chmod 124 i_02.png

Solución

Los permisos asignados son muy restrictivos. Por ejemplo, el propietario no tiene permiso de lectura ni de escritura, solo de ejecución, lo cual es muy inusual para un archivo de imagen.

Explicación

Los permisos en notación octal se leen de la siguiente manera:

  • 1 (Propietario): Solo ejecución (---x). No puede leer ni modificar su propio archivo.
  • 2 (Grupo): Solo escritura (--w-). Los miembros del grupo pueden modificarlo, pero no leerlo.
  • 4 (Otros): Solo lectura (r--). Cualquier otro usuario puede leerlo.

Problema 7

¿Qué comandos en Linux realiza un listado completo de los archivos cuyo nombre comience con la letra D y la extensión tenga 3 caracteres exactamente?

Solución

ls -l D*.???

Explicación

  • ls: Comando para listar archivos.
  • -l: Opción para mostrar un listado largo y detallado.
  • D*: Patrón que coincide con cualquier nombre que empiece con "D". El asterisco (*) es un comodín para cero o más caracteres.
  • .???: Patrón que coincide con cualquier extensión de exactamente 3 caracteres. El signo de interrogación (?) es un comodín para un solo carácter.

Problema 8

¿Qué tipo de autorización se le asigna al archivo trabajo.txt con el siguiente comando: chmod 764 trabajo.txt?

Solución

  • Propietario: Lectura, escritura y ejecución (rwx).
  • Grupo: Lectura y escritura (rw-).
  • Otros: Solo lectura (r--).

Explicación

La notación octal es la suma de los valores de los permisos (Lectura=4, Escritura=2, Ejecución=1):

  • 7 = 4 + 2 + 1 (rwx)
  • 6 = 4 + 2 (rw-)
  • 4 = 4 (r--)

Problema 9

Indique la acción que realiza el siguiente comando pwd.

Solución

Muestra la ruta completa del directorio de trabajo actual.

Explicación

pwd son las siglas de "Print Working Directory" (Imprimir Directorio de Trabajo). Es útil para saber en qué parte del sistema de archivos te encuentras.

Problema 10

¿Qué realiza la instrucción ls | more?

Solución

Lista el contenido del directorio y muestra el resultado de forma paginada, es decir, una pantalla a la vez.

Explicación

La salida del comando ls es "redirigida" a través de la tubería (|) como entrada para el comando more, el cual permite visualizar la información por páginas, avanzando con la barra espaciadora.

Problema 11

Indique la acción que realiza el siguiente comando grep.

Solución

Busca líneas que contengan un patrón de texto específico dentro de uno o más archivos.

Explicación

grep es una herramienta muy potente para filtrar texto. Por ejemplo, grep "error" sistema.log buscaría y mostraría todas las líneas que contienen la palabra "error" en el archivo sistema.log.

Problema 12

Cuál es la instrucción correcta para que el propietario del archivo Texto pueda leer y escribir en el archivo.

Solución

chmod u+rw Texto

Explicación

chmod modifica los permisos. La letra u se refiere al propietario (user). El signo + añade permisos, y rw especifica lectura (read) y escritura (write). Este comando añade los permisos de lectura y escritura para el dueño sin alterar los demás permisos existentes.

Problema 13

¿Cómo se llama al núcleo del sistema operativo Linux?

Solución

Kernel.

Explicación

El Kernel es el núcleo o corazón del sistema operativo. Se encarga de gestionar los recursos del hardware y de permitir la comunicación entre los componentes de software y hardware.

Problema 14

¿Qué número determina los permisos de un archivo en un sistema operativo Linux?

Solución

La notación octal (números del 0 al 7).

Explicación

Se utiliza un código de tres dígitos en base 8 (octal), donde cada dígito representa los permisos para el propietario, el grupo y otros, respectivamente. Cada dígito es la suma de los valores: 4 (lectura), 2 (escritura) y 1 (ejecución).

Problema 15

Cuál es el comando para ordenar líneas de archivos de texto en Linux.

Solución

sort

Explicación

El comando sort lee las líneas de un archivo de texto, las ordena alfabética o numéricamente, y muestra el resultado en la salida estándar.

Problema 16

Indique las principales diferencias entre los siguientes comandos que sirven para la visualización de archivos: cat, more, less, head, tail.

Solución y Explicación

  • cat: Muestra el contenido completo de un archivo de una sola vez, sin pausas. Es ideal para archivos cortos.
  • more: Muestra el contenido del archivo página por página. Solo permite avanzar hacia adelante en el texto.
  • less: Es una versión mejorada de more. También muestra el contenido por páginas, pero permite avanzar y retroceder, además de realizar búsquedas.
  • head: Muestra las primeras 10 líneas de un archivo por defecto.
  • tail: Muestra las últimas 10 líneas de un archivo por defecto. Es muy útil para ver los registros más recientes de un archivo de log.

Problema 17

En Linux, en su directorio actual de trabajo cree el archivo Datos.dat con la información proporcionada.

Solución

echo -e "455613:Mamani:Aliaga:Tobias:UMSA:La Paz\n145673:Gutierrez:Pacheco:Sonia:USFA:La Paz\n789613:Paz:Cordero:Pedro Armando:UMSS:Cochabamba\n454589:Choque:Nina:Ana Luz:UMSA:La Paz\n102513:Rojas:Quispe:Pamela:UMSS:Cochabamba\n789456:Mamani:Ticona:Julio Cesar:UMSA:La Paz\n258963:Diaz:Condori:Lusmila:UMSA:La Paz\n654789:Vargas:Tola:Alejandro Juan:EMI:La Paz" > Datos.dat

Explicación

El comando echo -e permite interpretar secuencias de escape como \n (nueva línea), lo que nos permite escribir todo el contenido multilínea en un solo comando. El operador > redirige esa salida de texto y la guarda en el archivo `Datos.dat`.

Problema 18

Cree la siguiente estructura y muestre un listado de los archivos y subdirectorios.

Solución

Para crear la estructura:

mkdir -p PREFA/{COM99/{DOS,LINUX,LATEX,HTML},INF99/{ALG,DF/{REPETITIVAS,DECISION}},MAT99,LIN99}

a) Mostrar un listado del directorio actual:

ls

b) Mostrar con toda la información, permisos, tamaño, fecha de creación:

ls -l

c) Mostrar ordenados por la fecha:

ls -lt

Explicación

Se utiliza mkdir -p para crear toda la jerarquía de directorios de una sola vez. Luego, se usa el comando ls con diferentes opciones para listar el contenido: sin opciones para un listado simple, -l para un formato largo y detallado, y -lt para ordenar por fecha de modificación.

Problema 19

¿Cuáles son las características del Sistema operativo Linux?

Solución y Explicación

  • Multiusuario: Varios usuarios pueden acceder a los recursos del sistema simultáneamente.
  • Multitarea: El sistema puede ejecutar varios procesos o aplicaciones al mismo tiempo.
  • Código Abierto (Open Source): Su código fuente es libremente accesible para ser estudiado, modificado y distribuido.
  • Seguridad: Tiene un sistema robusto de permisos para archivos y procesos.
  • Estabilidad: Es conocido por su capacidad de funcionar por largos períodos sin reiniciarse.

Problema 21

Crear la estructura de subdirectorios y archivos en el directorio actual y resolver los incisos.

Solución

Para crear la estructura de directorios:

mkdir -p prefas/{mate/{geometria,algebra},fisica,com99/{dos,linux,html}}

a) Crear el archivo dirección.txt dentro del subdirectorio geometría:

touch prefas/mate/geometria/direccion.txt

b) Buscar la dirección “miraflores”, en los archivos con extensión “txt” del directorio geometría:

grep "miraflores" prefas/mate/geometria/*.txt

c) Quitar los permisos de lectura al dueño y grupo sobre archivo ayuda.txt del directorio prefas: (Asumiendo que el archivo existe)

chmod ug-r prefas/ayuda.txt

Explicación

  • touch es un comando para crear un archivo vacío.
  • grep busca patrones de texto dentro de archivos.
  • chmod ug-r modifica permisos: u (dueño), g (grupo), - (quitar permiso), r (lectura).

Problema 22: Comando LS

Dado el comando LS, ¿cuáles son sus opciones? Presentar dos ejemplos por cada opción y explicar.

Opción: -l (long format)

Explicación: Muestra una lista detallada con permisos, propietario, grupo, tamaño, fecha de modificación y nombre.

  • Ejemplo 1: ls -l
    Realiza un listado largo del contenido del directorio actual.
  • Ejemplo 2: ls -l /etc/
    Realiza un listado largo del contenido del directorio /etc.

Opción: -a (all)

Explicación: Muestra todos los archivos y directorios, incluyendo los ocultos (aquellos cuyos nombres empiezan con un punto).

  • Ejemplo 1: ls -a
    Muestra todos los archivos del directorio actual, incluidos los ocultos.
  • Ejemplo 2: ls -la ~
    Muestra un listado largo y detallado de todos los archivos en el directorio home del usuario (~).

Problema 23: Comando CHMOD

Dado el comando CHMOD, ¿cuáles son sus opciones? Presentar dos ejemplos por cada opción y explicar.

Opción: -R (recursive)

Explicación: Aplica los cambios de permisos a un directorio y a todo su contenido (archivos y subdirectorios) de forma recursiva.

  • Ejemplo 1: chmod -R 755 public_html
    Establece permisos de lectura y ejecución para todos, y escritura solo para el dueño, a toda la carpeta public_html y su contenido.
  • Ejemplo 2: chmod -R go-w confidential
    Quita el permiso de escritura (w) para el grupo (g) y otros (o) a la carpeta "confidential" y todo lo que contiene.

Uso: Notación Simbólica (u, g, o, a)

Explicación: Permite modificar permisos de forma aditiva (+) o sustractiva (-) para el usuario (u), grupo (g), otros (o) o todos (a).

  • Ejemplo 1: chmod u+x mi_script.sh
    Añade (+) el permiso de ejecución (x) para el usuario/dueño (u) del archivo.
  • Ejemplo 2: chmod o-rwx secret.dat
    Quita (-) los permisos de lectura (r), escritura (w) y ejecución (x) para otros (o).

Problema 24: Comando SORT

Dado el comando SORT, ¿cuáles son sus opciones? Presentar dos ejemplos por cada opción y explicar.

Opción: -r (reverse)

Explicación: Invierte el resultado de la ordenación, mostrando los resultados en orden descendente.

  • Ejemplo 1: sort -r nombres.txt
    Ordena las líneas del archivo de la Z a la A.
  • Ejemplo 2: ls | sort -r
    Lista los archivos del directorio y los ordena en orden alfabético inverso.

Opción: -n (numeric sort)

Explicación: Realiza una ordenación numérica en lugar de una alfabética. Es esencial para ordenar listas de números correctamente.

  • Ejemplo 1: sort -n puntuaciones.txt
    Ordena un archivo que contiene números en cada línea de menor a mayor.
  • Ejemplo 2: du -h | sort -h
    La opción -h (human-readable) de sort se usa junto a comandos como `du -h` para ordenar tamaños de archivo como 1K, 2M, 1G correctamente.

Problema 25: Comando FIND

Dado el comando FIND, ¿cuáles son sus opciones? Presentar dos ejemplos por cada opción y explicar.

Opción: -name

Explicación: Busca archivos y directorios basándose en su nombre. Se pueden usar comodines como '*' entre comillas.

  • Ejemplo 1: find /home -name "*.pdf"
    Busca todos los archivos que terminen en .pdf dentro del directorio /home y todos sus subdirectorios.
  • Ejemplo 2: find . -name "config.txt"
    Busca un archivo llamado exactamente "config.txt" a partir del directorio actual (.).

Opción: -type

Explicación: Filtra la búsqueda por el tipo de archivo. Los más comunes son 'f' para archivo regular y 'd' para directorio.

  • Ejemplo 1: find /var/log -type f
    Encuentra únicamente los archivos (no directorios) dentro de la ruta /var/log.
  • Ejemplo 2: find . -type d -name "backups"
    Encuentra únicamente directorios que se llamen "backups" a partir de la ubicación actual.

Problema 26: Comando CAT

Dado el comando CAT, ¿cuáles son sus opciones? Presentar dos ejemplos por cada opción y explicar.

Opción: -n (number)

Explicación: Numera todas las líneas de la salida, incluyendo las líneas en blanco.

  • Ejemplo 1: cat -n mi_archivo.txt
    Muestra el contenido del archivo con cada línea precedida por su número.
  • Ejemplo 2: cat -n archivo1.txt archivo2.txt
    Concatena ambos archivos y muestra el resultado con una numeración de líneas continua.

Opción: -b (number-nonblank)

Explicación: Es similar a -n, pero numera únicamente las líneas que no están vacías.

  • Ejemplo 1: cat -b codigo.py
    Muestra el código fuente numerando solo las líneas que contienen código o comentarios, ignorando las líneas en blanco.
  • Ejemplo 2: cat -b poema.txt
    Muestra un poema numerando los versos pero no los espacios en blanco entre estrofas.
